1.4. INDIGOServerConnection¶

class INDIGOServerConnection(name, host, port)[source]¶

This is a class that represents a Server and has all the tools for the connection and management of it.

Variables:
  • name – Name of the server.
  • _host – IP address of the server.
  • _port – Port for listener of the server. In INDIGO the default port is 7624.
  • _sock – This is a variable that will contain a socket connection to the server.
  • _endReading – Check if the socket has reached the end of reading for the pipeline of the server connection.
  • _thread – This is a variable that will contain a thread that executes a _readerFunction concurrently with the main function.
  • devices – A dictionary of all devices of the server. Has the name of device as key and the INDIGODevice as value.
  • wait – Set a time to wait to execute some functions.
  • blobMode – It indicate if the BLOB mode is Never or URL.
  • propertyListeners – List of listeners for properties.
  • messageListeners – List of listeners for messages.
  • serverListener – List of listeners for the server.
_parseVectorTag(tag)[source]¶

Here we create the new elements by instantiating an INDIGODevice and calling its _parserVectorTag method. Finally, we get the name of the device and the name of the property to check if it is in the list of listeners and execute its listener. This function is private, you must not use it.

Parameters:tag (xml.etree.ElementTree.Element) – Element of the parser of XML message.
_parserMessageTag(tag)[source]¶

Here we parse the messages coming from the server. This function is private, you must not use it.

Parameters:tag (xml.etree.ElementTree.Element) – Element of the parser of XML message.
_readerFunction()[source]¶

In this function, we read all messages from the server and call some functions that implement the parser in each case. This is an infinte loop that only finish when the socket is closed. This function is private, you must not use it.

_send(command)[source]¶

This function is used to send messages to the server whenever needed. This message can be used to turn on or off some devices; create, modify or erase some property, etc. This function is private, you must not use it.

Parameters:command (str) – Message to send.

connect()[source]¶

In this function we create a connection with the server using a socket connection. Also, we launch a thread with the execution of _readerFunction to read all messages that the server sends every time. And finally, send a message to the server to get all the properties that it has.

downloadImage(path)[source]¶

We create a URL with the path in the parameter, the IP of the host, and the port. Later, we check if the downloaded path exists, and if it doesn’t exist, we create it. Then we save the image file on the client device and show the image.

Parameters:path (str) – Path of the image file in the server.
enableBLOB()[source]¶

To take photos and get them, we need to activate the BLOBs on the server. This function changes the value of BLOB to URL, since this value is NEVER by default.

sendBLOBMessage(device, property)[source]¶

This function sends a message to the server to enable the BLOB for a specific property. To do this, the property must be of BLOB type. You must call this function if you want to enable BLOB correctly, as the enableBLOB function doesn’t work on its own.

Parameters:

  • property (str.) – Name of the property that will have enabled or disabled the BLOB.
